# Video Type: Tutorial

## Overview

Step-by-step instructional videos that teach viewers how to accomplish a specific task in software or tools. Tutorials guide the viewer through a process with clear, sequential instructions.

## Characteristics

| **Aspect** | **Value** |
|------------|-----------|
| **Duration** | 3-15 minutes |
| **Pacing** | Slow, deliberate, step-by-step |
| **Tone** | Patient, encouraging, clear |
| **WPM** | 120-140 (slower for complex steps) |
| **Scene Changes** | Every 30-90 seconds |
| **Primary Visual** | Screencast + Avatar |

## Structure Template

1. **Hook** (5-10s) — State what the viewer will learn
2. **Prerequisites** (10-20s) — What they need before starting
3. **Step 1-N** (30-90s each) — Individual steps with clear narration
4. **Verification** (10-20s) — How to confirm it worked
5. **Summary** (10-15s) — Recap key steps
6. **Next Steps / CTA** (5-10s) — What to do next

## Script Conventions

- Use imperative mood: "Click on...", "Open the..."
- Number each step explicitly: "In Step 1, we'll..."
- Pause after each step (2-3 seconds) for viewer to follow along
- Always mention what the expected result should be
- Avoid jargon without explanation
- Max 20 words per sentence for narration clarity

## Visual Direction

- **Primary:** Split screen (avatar left, screencast right) or screencast with avatar overlay
- **Steps:** Highlight/zoom on relevant UI elements
- **Transitions:** Simple cuts between steps, fade for section changes
- **On-Screen Text:** Step numbers, key commands, important values
- **Screenshots:** Capture BEFORE and AFTER state of each step

## Pacing Rules

- Max 8 words of on-screen text at once
- Pause 2-3 seconds between major steps
- No scene longer than 90 seconds
- Include visual indicators (arrows, highlights) for every click/action
- Total word count: 400-2000 words (scales with duration)

## Anti-Patterns

- Rushing through steps without pausing
- Assuming prior knowledge without stating prerequisites
- Skipping error states or edge cases
- Too much talking without visual support
- Missing verification step at the end

## Example Storyboard

```
Scene 1 (10s): Avatar — "In this tutorial, you'll learn how to install the Gallery module for OXID eShop."
Scene 2 (15s): Screencast — Show terminal, type composer command, highlight output
Scene 3 (20s): Split — Avatar explains what the command does while screencast shows progress
Scene 4 (30s): Screencast — Navigate to admin panel, show module activation
Scene 5 (15s): Screencast — Verify module appears in module list
Scene 6 (10s): Avatar — "That's it! The Gallery module is now active."
```
