# Video Type: Training

## Overview

Structured educational videos designed to teach specific skills or knowledge. Training videos are longer, more thorough, and often part of a series or learning path. They include learning objectives, practical exercises, and knowledge checks.

## Characteristics

| **Aspect** | **Value** |
|------------|-----------|
| **Duration** | 5-20 minutes |
| **Pacing** | Structured, moderate, with deliberate pauses |
| **Tone** | Professional, patient, encouraging |
| **WPM** | 130-140 (comprehension priority) |
| **Scene Changes** | Every 30-60 seconds |
| **Primary Visual** | Mixed (avatar + screencast + slides) |

## Structure Template

1. **Learning Objectives** (15-20s) — What the viewer will be able to do after watching
2. **Context** (20-30s) — Why this topic matters, where it fits in the bigger picture
3. **Theory/Concept** (60-120s) — Core knowledge explained with visuals
4. **Demonstration** (120-300s) — Practical walkthrough with screen recording
5. **Practice Exercise** (30-60s) — Prompt for viewer to try it themselves
6. **Common Mistakes** (30-60s) — What to watch out for
7. **Knowledge Check** (20-30s) — Quick quiz or self-assessment question
8. **Summary** (15-20s) — Recap learning objectives
9. **Next Steps** (10-15s) — What to learn next in the series

## Script Conventions

- Start with learning objectives: "After this video, you'll be able to..."
- Use the "Tell, Show, Do" pattern: explain → demonstrate → prompt practice
- Signpost navigation: "In the next section, we'll cover..."
- Repeat key terms: introduce, use in context, summarize
- Use "we" language: "Let's open the admin panel..."
- Include pause prompts: "Pause the video here and try this yourself."
- Avoid assumptions: check prerequisites at the start

## Visual Direction

- **Theory Sections:** Slides with diagrams, flowcharts, or key term highlights
- **Demo Sections:** Full-screen screencast with avatar overlay (small, bottom-right)
- **Practice Prompts:** Slide with exercise instructions, timer icon
- **Knowledge Checks:** Slide with question + answer reveal after pause
- **Section Breaks:** Brief title card between major sections
- **Consistent Layout:** Same visual template throughout the entire series

## Pacing Rules

- Theory: 130 WPM (slower for complex concepts)
- Demo: 120 WPM (viewer needs to follow along)
- Summary: 140 WPM (viewer already knows the content)
- Pause 3-5 seconds after key concepts
- Include "pause and try" prompts every 3-5 minutes
- Total word count: 700-2800 words

## Anti-Patterns

- No learning objectives stated
- All theory, no practical demonstration
- Demo too fast for viewer to follow
- No practice or self-check opportunity
- Mixing multiple topics in one video (split into episodes)
- Inconsistent visual style across a series
- No signposting (viewer doesn't know where they are)

## Example Storyboard

```
Scene 1 (15s): Slides — "Learning Objectives: 1. Configure product gallery 2. Upload images 3. Set display options"
Scene 2 (25s): Avatar — Context: why image presentation matters for conversion
Scene 3 (60s): Slides — Theory: image formats, resolution, file size considerations
Scene 4 (45s): Screencast — Demo: navigate to admin panel, open gallery config
Scene 5 (90s): Screencast — Demo: upload images, drag to reorder, set alt text
Scene 6 (20s): Slides — "Pause here and try uploading 3 test images to your gallery"
Scene 7 (45s): Screencast — Demo: configure display settings (grid, lightbox, zoom)
Scene 8 (30s): Avatar — Common mistakes: wrong format, missing alt text, oversized files
Scene 9 (20s): Slides — "Quick Check: What's the recommended max file size?"
Scene 10 (15s): Avatar — Summary and next episode preview
```
