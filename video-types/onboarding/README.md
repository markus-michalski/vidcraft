# Video Type: Onboarding

## Overview

Welcome and orientation videos for new users, customers, or employees. Onboarding videos reduce support tickets and time-to-value by guiding people through their first steps in a friendly, non-overwhelming way.

## Characteristics

| **Aspect** | **Value** |
|------------|-----------|
| **Duration** | 3-10 minutes |
| **Pacing** | Friendly, clear, unhurried |
| **Tone** | Warm, welcoming, supportive |
| **WPM** | 130-140 |
| **Scene Changes** | Every 20-40 seconds |
| **Primary Visual** | Avatar + screencast (guided tour) |

## Structure Template

1. **Welcome** (10-15s) — Greet and set expectations
2. **What You'll Get** (15-20s) — Overview of value and capabilities
3. **First Steps** (60-120s) — Account setup, initial configuration, or workspace tour
4. **Core Feature 1** (30-60s) — The most important thing they'll use
5. **Core Feature 2** (30-60s) — Second most important feature
6. **Quick Win** (30-45s) — Help them achieve something immediately
7. **Getting Help** (15-20s) — Where to find docs, support, community
8. **What's Next** (10-15s) — Next video in series or next milestone

## Script Conventions

- First sentence must be welcoming: "Welcome to [product]!"
- Use "you" constantly: "Your dashboard shows...", "You can..."
- Celebrate the viewer's progress: "Great, you've already set up..."
- Limit to 2-3 features (don't overwhelm on day one)
- Always point to help resources
- End with encouragement: "You're all set to get started!"

## Visual Direction

- **Avatar:** Warm, friendly presence — appears frequently (more than other types)
- **Screencast:** Guided tour style — cursor moves deliberately, highlights clickable areas
- **Background:** Bright, clean, welcoming (not dark/corporate)
- **On-Screen Text:** Friendly labels, step numbers, celebration messages
- **Transitions:** Smooth, gentle (no abrupt cuts)
- **Progress Indicator:** Optional progress bar or step counter

## Pacing Rules

- Never rush — new users need time to orient
- One concept per scene (don't combine features)
- Pause 2 seconds after each "now click here" instruction
- On-screen text for every button name and menu path
- Quick win within first 3 minutes (instant gratification)
- Total word count: 400-1400 words

## Anti-Patterns

- Starting with features instead of welcome/context
- Covering too many features (max 3 for onboarding)
- Assuming prior knowledge of the product
- No clear "quick win" moment
- Missing help/support section
- Cold, corporate tone (this is their first impression!)
- Too long (>10 min loses new users who are already overwhelmed)

## Example Storyboard

```
Scene 1 (12s): Avatar — "Welcome to your new OXID eShop! I'll help you get set up in the next 5 minutes."
Scene 2 (18s): Slides — "What you'll be able to do: 1. Add products 2. Configure your theme 3. Process your first order"
Scene 3 (45s): Screencast — Login to admin, tour of main navigation (sidebar, top bar, dashboard)
Scene 4 (50s): Screencast — Add your first product: title, price, image, save
Scene 5 (30s): Avatar + Screencast — "Let's see it live!" → Show product on frontend
Scene 6 (15s): Avatar — "Congratulations! Your first product is live."
Scene 7 (15s): Slides — "Need help? Documentation at docs.example.com, Support at support@example.com"
Scene 8 (10s): Avatar — "Next up: configuring your shop's appearance. See you in the next video!"
```
