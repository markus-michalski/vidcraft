# Video Type: Explainer

## Overview

Short, dynamic videos that explain a concept, product, or process in a digestible format. Explainers distill complex topics into clear, engaging narratives that hold viewer attention through visual variety and benefit-focused messaging.

## Characteristics

| **Aspect** | **Value** |
|------------|-----------|
| **Duration** | 60-120 seconds |
| **Pacing** | Medium to fast, dynamic |
| **Tone** | Clear, engaging, benefit-focused |
| **WPM** | 140-150 |
| **Scene Changes** | Every 10-20 seconds |
| **Primary Visual** | Mixed (avatar + slides + graphics) |

## Structure Template

1. **Hook** (5-8s) — Problem statement or surprising fact
2. **Problem** (10-15s) — Expand on the pain point, make it relatable
3. **Solution** (15-20s) — Introduce the solution (your product/concept)
4. **How it Works** (20-30s) — 2-3 key mechanisms or features
5. **Proof/Result** (10-15s) — Evidence, testimonial, or metric
6. **CTA** (5-10s) — Clear next step

## Script Conventions

- Start with a question or bold statement, never "In this video..."
- One idea per sentence, one concept per scene
- Use analogies for complex concepts: "Think of it like..."
- Benefit before feature: "Save 3 hours" before "automatic batch processing"
- Use numbers for credibility: "87% of users...", "in under 2 minutes"
- End with ONE clear CTA, not multiple options

## Visual Direction

- **Primary:** High visual variety — alternate between avatar, graphics, text animations
- **Never** more than 15 seconds on one visual without change
- **Graphics:** Icons, diagrams, process flows (simple, clean)
- **Text Animations:** Key stats and quotes as animated text overlays
- **Avatar:** Appears at hook and CTA, minimal during explanation
- **Transitions:** Smooth, fast transitions (slides, fades) — no hard cuts

## Pacing Rules

- Scene change every 10-20 seconds (faster than tutorials)
- Max 5-6 scenes total (keep it tight)
- On-screen text: max 5 words per overlay
- No filler content — every second must earn its place
- Total word count: 150-300 words
- If you can't explain it in 120 seconds, it's not an explainer

## Anti-Patterns

- Starting with "Welcome to..." or "In this video..."
- Feature dumping instead of benefit highlighting
- More than 3 key points (viewer can't retain more)
- Talking head for entire video (need visual variety)
- Weak or missing CTA
- Over 2 minutes (it's not an explainer anymore, it's a tutorial)

## Example Storyboard

```
Scene 1 (6s): Text animation — "Managing shop images takes too long."
Scene 2 (12s): Avatar — "What if you could upload, optimize, and display product images in one step?"
Scene 3 (8s): Slides — Icon: Upload → Optimize → Display (animated flow)
Scene 4 (25s): Screencast — Quick demo of drag-and-drop, auto-optimization, gallery result
Scene 5 (12s): Slides — "87% faster image workflow" with checkmark animation
Scene 6 (8s): Avatar — "Try it free for 30 days. Link below."
```
