# Video Type: FAQ Video

## Overview

Focused, single-question videos that provide direct answers to common customer questions. Each video addresses exactly one question with a clear, authoritative response and relevant context. Designed for embedding on FAQ pages, knowledge bases, or support portals.

## Characteristics

| **Aspect** | **Value** |
|------------|-----------|
| **Duration** | 30-90 seconds per question |
| **Pacing** | Direct, efficient, no filler |
| **Tone** | Helpful, authoritative, concise |
| **WPM** | 140-150 (clear and steady) |
| **Scene Changes** | Every 15-30 seconds |
| **Primary Visual** | Avatar + Supporting Screenshots/Diagrams |

## Structure Template

1. **Question** (5-10s) — State the exact question clearly
2. **Answer** (15-40s) — Direct answer first, then explain
3. **Context** (10-25s) — Additional details, edge cases, or related info
4. **Wrap-Up** (5-10s) — Summary sentence + pointer to related resources

## Script Conventions

- State the question verbatim as viewers would ask it
- Answer in the first sentence — never bury the lead
- Use "inverted pyramid" style: most important info first
- One question per video — never combine multiple questions
- If the answer has steps, keep them to 3 or fewer
- Avoid hedging language: say "This happens because..." not "This might happen because..."
- Max 18 words per sentence for clarity

## Visual Direction

- **Primary:** Avatar delivers answer directly to camera
- **Question Display:** Show the question as on-screen text at the start
- **Supporting Visuals:** Screenshots, simple diagrams, or UI highlights for context
- **Transitions:** Simple cuts, minimal motion — clarity over style
- **On-Screen Text:** The question, key terms, and any referenced URLs or paths

## Pacing Rules

- Max 10 words of on-screen text at once
- No scene longer than 30 seconds
- Question must appear on screen within first 3 seconds
- No intro music or branding before the question
- Total word count: 70-200 words (scales with duration)

## Anti-Patterns

- Answering multiple questions in one video
- Long introductions before getting to the answer
- Vague answers that don't resolve the question
- Using technical jargon without explanation
- Adding unnecessary filler ("Great question!", "Many people ask...")
- Ending without a clear resolution

## Example Storyboard

```
Scene 1 (5s): Text overlay + Avatar — "How do I reset my admin password?"
Scene 2 (20s): Avatar + Screencast — "Go to the login page, click 'Forgot Password', and enter your email. You'll receive a reset link within 2 minutes."
Scene 3 (15s): Screencast — Show the exact steps on screen, highlight the reset link in the email
Scene 4 (5s): Avatar — "If the email doesn't arrive, check your spam folder or contact support."
```
